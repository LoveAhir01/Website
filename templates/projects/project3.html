Nutritional Information Manager

Objective

The Nutritional Information Manager is a web-based application that enables users to track their daily nutritional intake and create personalized meal plans. It offers insights into calorie consumption, macronutrient distribution, and meal planning.

Technologies Used

  • Frontend: JavaScript, HTML, CSS
  • Backend: Flask (Python)
  • Database: PostgreSQL

Challenges

One of the main challenges was designing a dynamic system that allowed users to add new food items and calculate their nutritional values in real-time. Ensuring a smooth user experience with responsive design was also critical.

Process

The project was executed in the following phases:

  1. Design: Created a relational database schema to store food item details, user profiles, and daily intake logs.
  2. Backend Development: Built REST APIs using Flask for user authentication, food data management, and real-time calculations of nutritional values.
  3. Frontend Development: Designed a responsive user interface with JavaScript for dynamic updates and intuitive interaction.

Results

  • Enabled users to track their nutritional intake with 90% satisfaction rate.
  • Reduced meal planning time by 50% through automation.
  • Provided detailed nutritional insights, helping users adopt healthier eating habits.

Team Contribution

This was an individual project where I designed, developed, and tested the application independently. It allowed me to gain hands-on experience with Flask, database design, and frontend development.
