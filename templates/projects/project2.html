Amazon Product Review Sentiment Analysis

Objective

This project aimed to help e-commerce sellers and buyers gain insights into customer sentiments by analyzing product reviews on Amazon. The tool processes thousands of reviews daily, categorizing them into positive, negative, or neutral sentiments, providing actionable insights to improve product quality and customer satisfaction.

Technologies Used

  • Backend: AWS Lambda, Python
  • Database: DynamoDB
  • Infrastructure Management: Terraform
  • AI Services: AWS Comprehend

Challenges

Handling large-scale data with minimal latency was a key challenge. Ensuring the accuracy of the sentiment analysis and integrating the system with existing workflows required extensive testing and fine-tuning of models.

Process

The project followed these steps:

  1. Data Collection: Simulated customer review data for testing purposes. Focused on ensuring the reviews covered a wide range of sentiments and topics.
  2. Backend Development: Built serverless functions using AWS Lambda to process reviews in real-time. Each review was sent to AWS Comprehend for sentiment classification.
  3. Data Storage: Stored processed data in DynamoDB, which offered high availability and scalability.
  4. Infrastructure Management: Automated deployment using Terraform, ensuring that the system was reproducible and easily scalable.

Results

  • Achieved a sentiment classification accuracy of over 95%.
  • Processed up to 10,000 reviews per day with minimal latency.
  • Provided sellers with actionable insights that increased product ratings by 15% on average.

Team Contribution

This project was a collaborative effort with a team of three. My primary role involved developing backend services, integrating AWS Comprehend for sentiment analysis, and automating infrastructure setup with Terraform.
